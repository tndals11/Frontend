# ⚛️ React: 기본 원리부터 Hook의 탄생까지

이 문서는 React의 핵심적인 동작 방식, 기본 구성 요소, 그리고 **`props`의 한계를 극복하기 위해 Hook(useState, useRef, useEffect)이 등장한 배경**을 정리한 학습 요약본입니다.

---

## 1. ✨ React 개요 및 사용 가치

React는 사용자 인터페이스(UI)를 구축하는 **JavaScript 라이브러리**입니다.

| 특징 | 핵심 역할 |
| :--- | :--- |
| **선언적** (Declarative) | UI의 **최종 상태**만 정의하면, React가 알아서 업데이트를 처리하여 코드가 예측 가능합니다. |
| **컴포넌트 기반** (Component-Based) | UI를 **재사용 가능한 부품**으로 나누어 개발합니다. (개발/유지보수성 극대화) |
| **가상 DOM** (Virtual DOM) | 실제 DOM 대신 메모리상의 가상 복사본을 비교하여 **최소한의 변경**만 반영해 성능을 최적화합니다. |

---

## 2. 🧱 기본 구성 요소 및 데이터 원칙

### A. 컴포넌트와 JSX

* **함수형 컴포넌트**: **최신 React 표준**이며, 순수 JavaScript 함수로 작성되고 **Hook**을 사용해 상태와 기능을 관리합니다.
* **JSX**: JavaScript 확장 문법으로, 코드 내부에 **HTML과 유사한 마크업**을 작성하여 UI 구조를 직관적으로 만듭니다.

### B. 데이터 흐름의 두 축: Props와 State

| 원칙 | 특징 | 역할 |
| :--- | :--- | :--- |
| **Props** (Properties) | **🛑 읽기 전용 (Read-only)** | 부모에서 자식으로 데이터를 전달하는 **단방향 통로**입니다. |
| **State** (상태) | **변경 가능** (Mutable) | 컴포넌트 내부에서 관리되며, 변경 시 **자동으로 다시 렌더링**됩니다. |

---

## 3. 🔑 Props의 한계와 3대 Hook의 등장 배경

**`props`가 불변(읽기 전용)** 이라는 원칙 때문에, 컴포넌트 내부에서 데이터를 변경하거나 외부 작업을 처리하는 기능이 필요해졌고, 이를 위해 Hook이 탄생했습니다.

### 3.1. `useState`: **변경 가능성 확보 (Mutability)**

`props`가 불가능했던 **데이터 변경과 UI 업데이트**를 가능하게 합니다.

| 항목 | 역할 | Hook이 필요한 이유 |
| :--- | :--- | :--- |
| **핵심 역할** | **변경 가능한 상태(State)** 를 저장/관리합니다. | **읽기 전용**인 `props`와 달리, 사용자 상호작용에 따라 **데이터를 변경**하고 **화면에 반영**하기 위해 필요합니다. |
| **특징** | 상태가 변경되면 **자동으로 렌더링을 유발**합니다. | |

### 3.2. `useRef`: **렌더링 없는 값 저장 및 DOM 접근**

`useState`가 렌더링을 유발하는 문제를 해결하고, UI와 무관한 데이터를 다루기 위해 등장했습니다.

| 항목 | 역할 | Hook이 필요한 이유 |
| :--- | :--- | :--- |
| **핵심 역할** | 렌더링과 무관하게 값이 유지되는 **가변 값 저장** 및 **DOM 요소 접근**. | `useState`와 달리, 값이 변경되어도 **렌더링을 유발하지 않음**으로써 성능을 최적화하고, DOM 조작 기능을 제공합니다. |

### 3.3. `useEffect`: **외부 상호작용 (Side Effect) 안전한 실행**

컴포넌트의 순수한 렌더링 과정 바깥의 외부 세계와의 상호작용을 담당합니다.

| 항목 | 역할 | Hook이 필요한 이유 |
| :--- | :--- | :--- |
| **핵심 역할** | 컴포넌트 **렌더링이 완료된 후**에 **부수 효과 (Side Effect)** 를 처리합니다. | API 호출, 타이머 설정 등 비동기 작업을 렌더링에 방해 없이 수행하고, **의존성 배열**을 통해 **무한 루프(Infinite Loop)** 를 방지하기 위함입니다. |

## 4. 🔗 기타 주요 Hook 및 생태계

| Hook | 목적 | 환경/생태계 | 설명 |
| :--- | :--- | :--- | :--- |
| **`useContext`** | **Prop Drilling** 문제 해결 | **Next.js / Gatsby** | 서버 사이드 렌더링 (SSR) 등을 지원하는 React 기반 프레임워크 |
| **`useReducer`** | 복잡한 상태 로직 관리 | **Vite** | 가벼운 React 개발 환경 구성 도구
